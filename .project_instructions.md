# CashFlowIQ MVP - Project Instructions

## Project Overview
CashFlowIQ is an AI-powered SaaS cash flow forecasting tool for SMBs. This MVP includes:
- Landing page with pricing tiers (Solo $19/mo, Pro $59/mo, Team $149/mo)
- Email/password authentication with profile onboarding
- Dashboard with cash flow metrics and 90-day forecast
- Transactions management (CRUD operations)
- Reports with data visualizations (trend, category, monthly)
- Settings (profile, subscription, integrations)
- Stripe billing integration for subscription management

## Technical Stack
- Framework: Next.js 14 (App Router) + React 18
- Database: PostgreSQL with Prisma ORM
- Auth: NextAuth.js v4 with Prisma adapter
- Billing: Stripe Checkout (subscription mode) + Billing Portal
- Charts: Recharts for visualizations
- UI: shadcn/ui components + Tailwind CSS 3
- Icons: Lucide React

## Design System
- Color scheme: Primary blue (#2563eb), Secondary green (#059669), Neutral grays
- Typography: DM Sans (body), Plus Jakarta Sans (display), JetBrains Mono (mono)
- Spacing: 8px grid base
- Components: Use STYLE_GUIDE.md components (Button, Card, Table, Badge, etc.)
- Animations: Framer Motion for scroll-triggered and interactive animations
- Responsive: Mobile-first with tablet/desktop breakpoints

## Database Schema
Key entities:
- User: id, email, name, businessName, businessType, subscriptionTier, createdAt
- Transaction: id, userId, type (income/expense), amount, category, date, description
- Forecast: id, userId, date, projectedCashFlow, confidence
- StripeCustomer: userId, stripeCustomerId, subscriptionId, currentPlan, renewalDate

## Authentication Flow
1. Landing page (logged-out view) with Sign Up / Log In CTAs
2. Signup: Email, password, name, businessName, businessType → creates user + onboarding
3. Login: Email/password authentication
4. Dashboard: Protected route, redirects unauthenticated to login
5. Settings: Profile editing, subscription management via Stripe portal

## Stripe Billing Integration
- Use Stripe Checkout in subscription mode (not Payment Links, not deprecated redirectToCheckout)
- Products created with prices: Solo ($19/mo), Pro ($59/mo), Team ($149/mo)
- Webhook handling: checkout.session.completed, invoice.paid, invoice.payment_failed
- Billing Portal: For subscription management, payment method updates, invoices
- Never expose secret keys; use publishable key on client
- Dynamic URLs via request.headers.get("origin") for success/cancel redirects

## Key Implementation Notes
- Server components by default; use "use client" only for interactive elements
- Hooks (useState, useEffect) ONLY in client components
- Use Prisma queries with proper error handling
- Sample data: Realistic transactions and forecasts for demo account
- All passwords hashed with bcryptjs
- Session persistence across refreshes
- Responsive design: Mobile, tablet, desktop
- Animations: Framer Motion for scroll effects and transitions
- Hydration-safe: No non-deterministic functions in SSR markup

## Design Decisions
- Landing page serves logged-out users; dashboard is protected
- Sidebar navigation for app pages (Dashboard, Transactions, Reports, Settings)
- Cards with shadows and hover effects for visual hierarchy
- Charts use Recharts with responsive containers
- Forms use react-hook-form for validation
- Modals for adding/editing transactions
- Color-coded transaction types (income green, expense red)
- Real sample data on signup for immediate usability
